# 岗位评分框架

## 评分维度（满分100分）

| 维度 | 默认权重 | 说明 |
|------|---------|------|
| 薪资匹配度 | 18分 | 对比用户的 Base 底线 |
| 公司规模与规范度 | 20分 | 融资/规模/管理体系 |
| 技术成长空间 | 15分 | 与用户技术方向的匹配度和成长性 |
| 工作节奏与心理适配度 | 15分 | 加班/管理风格/心理安全感 |
| 稳定性 | 20分 | 资金来源判断 |
| 区域居住可行性 | 12分 | 区域生活质量 |
| **总计** | **100分** | |

> 用户可在 `user_profile.md` 中自定义各维度权重。修改后确保总分 = 100 分即可。

---

### 1. 薪资匹配度（默认权重18分）

| 分值 | 条件 |
|------|------|
| 满分 | Base ≥ [用户的期望薪资] |
| 高分 | Base 在期望薪资和底线之间，公司稳定性高 |
| 中分 | Base 在底线附近，公司稳定性一般 |
| 低分 | Base 接近但略高于底线 |
| 0分 | Base < [用户的底线] |

**核心逻辑：**
- **稳定公司可 trade-off 薪资**：央企/国企/知名外企/上市公司，Base 接近底线时可接受更高分
- **不稳定公司必须高薪**：初创/A轮/B轮，Base 必须显著高于底线才能拿高分
- Base 薪资是核心指标，年终奖/期权/股票不作为达标条件，仅作参考加分。
- 具体阈值和 trade-off 规则从 `references/user_profile.md` 读取。

---

### 2. 公司规模与规范度（默认权重20分）

| 分值 | 条件 |
|------|------|
| 20 | 行业头部大厂或知名外企 |
| 15 | 中型上市公司、B轮+且融资额>1亿、成熟企业科技子公司 |
| 10 | B轮+、A轮但团队>50人、有明确产品线 |
| 5 | A轮及以下、团队<50人、管理可能混乱 |
| 0 | 天使轮/初创、无法确认规模 |

**判断依据：** 融资轮次、员工规模、是否有公开的制度和流程、是否有明确的技术管理体系。

---

### 3. 技术成长空间（默认权重15分）

| 分值 | 条件 |
|------|------|
| 满分 | 核心业务线，深入用户技术方向的核心（架构设计/框架开发），能建立个人技术护城河 |
| 高分 | 匹配用户技术方向但偏应用层，有一定深度但偏工程交付 |
| 中分 | 技术栈匹配但成长有限（维护型、工具平台定制开发） |
| 低分 | 技术栈部分匹配、主要工作为 CRUD |
| 0分 | 技术栈不匹配、纯外包/维护 |

**关键区分：**
- **深度成长**：能深入技术方向的核心机制，离开公司依然有竞争力
- **应用成长**：在技术方向上积累经验，但更多是工程交付能力
- **工具型成长**：熟练使用某平台/框架，但能力绑定在特定工具上

> 用户的技术方向和成长偏好从 `references/user_profile.md` 读取。

---

### 4. 工作节奏与心理适配度（默认权重15分）

| 分值 | 条件 |
|------|------|
| 满 | 明确双休、弹性工作、无996文化，管理风格尊重个体差异 |
| 高 | 基本双休、偶尔加班，管理风格偏中性，无明显的PUA/高压文化信号 |
| 中 | 大小周或节奏偏快，有中度考核压力，可能触发消耗感 |
| 低 | 996文化明确、严重内卷、高压KPI、管理风格专制 |

**心理适配度判断（参考用户画像中的心理模式）：**
- 高频质疑/否定型管理 → 高风险
- 绩效排名/末位淘汰 → 高风险
- 结果导向+过程自由 → 适配度高
- 微观管理/日报周报文化 → 中等风险，视强度而定

---

### 5. 稳定性（默认权重20分）

> **核心判断逻辑：** 账面盈利 ≠ 资金稳定。资金**来源**比账面**数字**更重要。

| 分值 | 条件 |
|------|------|
| 20 | 央企/国企/研究院 或 成熟外企大厂（资金来源：国家信用/全球调配） |
| 15 | 知名上市公司，主营业务盈利（资金来源：资本市场） |
| 10 | B轮+且融资额>1亿，资金充足 |
| 5 | A轮但融资额尚可，或商业模式尚在探索 |
| 0 | 天使轮/初创，资金消耗快 |

**资金来源优先级：**
```
国家信用（央企/国企）> 全球调配（外企）> 资本市场（上市公司）> 主营业务（私企）
```

**用户可自定义的 Trade-off 规则（在 user_profile.md 中设置）：**
- 稳定性≥[X]分（[稳定公司类型]）→ 可接受较低 Base
- 稳定性<[X]分（[不稳定公司类型]）→ 必须较高 Base

---

### 6. 区域居住可行性（默认权重12分）

> 核心逻辑：租房状态下通勤是可变的，公司地址决定住址。因此评估的是**公司所在区域是否适合居住**，而非通勤时间本身。

| 分值 | 条件 |
|------|------|
| 满分 | 周边租房性价比高，配套齐全，区域生活氛围好 |
| 高分 | 租房合理但配套一般，或社交氛围偏弱但有基本生活设施 |
| 中分 | 租房偏贵或配套较差，搬过去有隐形成本 |
| 低分 | 房租明显高于预算，或环境质量差 |
| 0分 | 区域完全不适合居住 |

## 评分结果判定

| 总分 | 判定 | 建议动作 |
|------|------|---------|
| 80-100 | ⭐ 强烈推荐 | 立即投递，优先准备面试 |
| 70-79 | ✅ 推荐投递 | 投递，认真准备面试 |
| 60-69 | ⚠️ 可以考虑 | 投递作为练手/保底，面试时重点考察短板维度 |
| 50-59 | 🔶 勉强可投 | 仅在无其他机会时投递，或当纯练手 |
| < 50 | ❌ 不建议 | 跳过，不符合底线条件 |

> 用户可在 `user_profile.md` 中自定义各分数段的判定标准。

---

## 重要程度评级

| 等级 | 条件 | 行动建议 |
|------|------|---------|
| S | 80分+ 且为目标行业头部公司核心团队 | 投入最多精力，深度准备 |
| A | 70-79分，或各维度较均衡 | 认真准备，标准面试准备 |
| B | 60-69分，各方面中等 | 适度准备，当练手机会 |
| C | <60分 | 面试前花10分钟看一眼JD，不额外准备 |

---

## 一票否决（默认）

以下情况无论总分多高，直接标记为 ❌：

- Base < [用户底线] 且无谈判空间（具体阈值从 user_profile.md 读取）
- 明确 996 或严重内卷文化
- 公司有严重负面新闻（大规模裁员欠薪等）
- 岗位职责与用户技术方向完全无关（纯运营、纯销售包装成技术岗）

> 用户可在 `user_profile.md` 中自定义或扩展一票否决条件。

---

## Trade-off 逻辑总结

```
稳定公司 → 薪资底线可适当下放
不稳定公司 → 薪资底线必须严格执行
既不稳定又低薪 → 一票否决
```

具体数值从 `references/user_profile.md` 读取。
